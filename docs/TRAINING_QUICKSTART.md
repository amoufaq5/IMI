# IMI Training Quick-Start Guide

> Practical guide covering foundation vs adapter training, IP ownership, MVP training,
> HuggingFace upload, and running/testing after training.

---

## 1. Foundation Model vs Adapter Training

### What Is Foundation Training?

Foundation training **adapts the entire Mixtral 8x7B base model** to the medical domain.
It teaches the model general medical knowledge, terminology, and reasoning patterns
using a large combined corpus (3M+ examples across all adapters).

```
Mixtral 8x7B (general purpose) ──[Foundation Training]──> Medical-Mixtral (medical domain)
```

| Aspect | Foundation Training | Adapter Training |
|--------|-------------------|------------------|
| **Purpose** | Teach general medical knowledge | Specialize for a user type |
| **Input data** | All datasets combined (3M+ examples) | Per-adapter datasets (50K-500K each) |
| **LoRA rank** | r=64, α=128 (larger = learns more) | r=32, α=64 (smaller = focused) |
| **Target layers** | q,k,v,o_proj **+ MoE experts (w1,w2,w3)** | q,k,v,o_proj only |
| **Training time** | ~12 hours on 1×A100-80GB | ~4-8 hours per adapter |
| **Output** | `models/foundation/` | `adapters/{name}/` |
| **Runs once?** | Yes, once (reuse for all adapters) | Once per adapter (6 total) |
| **Cost** | ~$1,500 | ~$300-400 per adapter |

### What Is Adapter Training?

Adapter training creates a **small, specialized LoRA module** (~50-100MB) that sits on top
of the foundation model. Each adapter tailors the model for a specific user type:

```
Medical-Mixtral ──[DPO Alignment]──> Safe-Medical-Mixtral
                                            │
                    ┌───────────────────────┼───────────────────────┐
                    ▼                       ▼                       ▼
            patient_triage          clinical_decision          education
              adapter                   adapter                 adapter
             (~80MB)                   (~80MB)                 (~80MB)
```

### What Is DPO Safety Alignment?

DPO (Direct Preference Optimization) is a **middle step** between foundation and adapter
training. It teaches the model to **prefer safe responses** over unsafe ones using
paired examples (chosen vs rejected).

```
Foundation Model ──[DPO with safety pairs]──> Safety-Aligned Model ──[LoRA]──> Adapters
```

### The Full Pipeline

```
Stage 1: Foundation     ─── "Learn medicine"        (~12 hrs, ~$1,500)
Stage 2: DPO Alignment  ─── "Learn to be safe"      (~2 hrs,  ~$200)
Stage 3: LoRA Adapters  ─── "Learn your user type"  (~4-8 hrs per adapter, ~$300 each)
```

## 3. MVP Training (Doctor + Patient Only)

### Why MVP First?

The MVP focuses on the two highest-value adapters:
- **clinical_decision** (Doctor) — Clinical decision support, drug interactions
- **patient_triage** (Patient) — Symptom assessment, triage, OTC guidance

These two cover the primary use cases for investor demos and early users.

### MVP Training Commands

```bash
# ── Step 1: Collect data (only doctor + patient datasets) ──
python scripts/data_collection/collect_datasets.py

# ── Step 2: Prepare data ──
python scripts/training/prepare_data.py --adapter clinical_decision
python scripts/training/prepare_data.py --adapter patient_triage

# ── Step 3: Foundation training (required, uses all data) ──
python scripts/training/train_foundation.py --max-examples 200000

# ── Step 4: DPO safety alignment ──
python scripts/training/train_dpo.py export
python scripts/training/train_dpo.py train --foundation-path models/foundation

# ── Step 5: Train only 2 adapters (parallel on 2 GPUs) ──
CUDA_VISIBLE_DEVICES=0 python scripts/training/train_lora.py \
    --adapter clinical_decision \
    --base-model models/dpo_aligned &

CUDA_VISIBLE_DEVICES=1 python scripts/training/train_lora.py \
    --adapter patient_triage \
    --base-model models/dpo_aligned &

wait
echo "MVP training complete!"

# ── Step 6: Evaluate both ──
python scripts/training/evaluate_adapter.py --adapter clinical_decision
python scripts/training/evaluate_adapter.py --adapter patient_triage
```

### MVP Cost Breakdown

| Step | GPU Hours | Cost (RunPod A100-80GB @ $2.49/hr) |
|------|-----------|-------------------------------------|
| Foundation training (200K examples) | ~6 hrs × 1 GPU | ~$15 |
| DPO alignment | ~2 hrs × 1 GPU | ~$5 |
| Doctor adapter | ~6 hrs × 1 GPU | ~$15 |
| Patient adapter | ~6 hrs × 1 GPU | ~$15 |
| Evaluation | ~1 hr × 1 GPU | ~$2.50 |
| **Total MVP** | **~21 GPU-hours** | **~$52** |

> **Note:** If training foundation on full 3M+ examples, foundation step rises to ~$30.
> Total MVP stays under **$100** on RunPod.

### MVP vs Full Training

| | MVP | Full |
|--|-----|------|
| Adapters | 2 (doctor + patient) | 6 (all user types) |
| Foundation examples | 200K | 500K+ |
| GPUs needed | 2×A100-80GB | 6×A100-80GB |
| Training time | ~12 hours | ~24 hours |
| Total cost | ~$50-100 | ~$200-500 |
| Use case | Demo, early users | Production |

## 4. Saving to HuggingFace After Training

### One-Time Setup

```bash
# Install HuggingFace CLI
pip install huggingface_hub

# Login (get token from https://huggingface.co/settings/tokens)
huggingface-cli login
```

### Upload Foundation Model

```bash
# Upload foundation weights to your HF org
huggingface-cli upload your-org/imi-medical-foundation models/foundation/ . \
    --commit-message "IMI Medical Foundation v1.0"
```

### Upload Individual Adapters

```bash
# Upload each adapter separately
huggingface-cli upload your-org/imi-patient-triage adapters/patient_triage/ . \
    --commit-message "Patient Triage Adapter v1.0"

huggingface-cli upload your-org/imi-clinical-decision adapters/clinical_decision/ . \
    --commit-message "Clinical Decision Adapter v1.0"

huggingface-cli upload your-org/imi-education adapters/education/ . \
    --commit-message "Education Adapter v1.0"
```

### Upload All Adapters at Once (Script)

```bash
#!/bin/bash
# save_all_to_hf.sh

ORG="your-org"  # Change to your HuggingFace org/username
VERSION="v1.0"

# Foundation
echo "Uploading foundation model..."
huggingface-cli upload ${ORG}/imi-medical-foundation models/foundation/ . \
    --commit-message "IMI Medical Foundation ${VERSION}"

# DPO-aligned model
echo "Uploading DPO-aligned model..."
huggingface-cli upload ${ORG}/imi-medical-dpo models/dpo_aligned/ . \
    --commit-message "IMI DPO Safety-Aligned ${VERSION}"

# Each adapter
for adapter in patient_triage clinical_decision clinical_pharmacist education regulatory_qa research; do
    if [ -d "adapters/${adapter}" ]; then
        echo "Uploading ${adapter}..."
        huggingface-cli upload ${ORG}/imi-${adapter} adapters/${adapter}/ . \
            --commit-message "${adapter} Adapter ${VERSION}"
    fi
done

echo "All uploads complete!"
```

### Make Repos Private (Recommended)

```bash
# Set repo visibility to private (do this on HuggingFace website or via API)
python -c "
from huggingface_hub import HfApi
api = HfApi()
api.update_repo_visibility('your-org/imi-medical-foundation', private=True)
api.update_repo_visibility('your-org/imi-patient-triage', private=True)
api.update_repo_visibility('your-org/imi-clinical-decision', private=True)
"
```

### Download Later (On New Machine)

```bash
# Download foundation
huggingface-cli download your-org/imi-medical-foundation --local-dir models/foundation

# Download a specific adapter
huggingface-cli download your-org/imi-patient-triage --local-dir adapters/patient_triage
```

## 5. Running & Testing After Training

### Option A: Quick Local Test (Single GPU)

```bash
# Test a single adapter interactively
python -c "
from scripts.training.evaluate_adapter import AdapterEvaluator

evaluator = AdapterEvaluator(
    base_model_name='models/dpo_aligned',  # or 'mistralai/Mixtral-8x7B-Instruct-v0.1'
    adapter_path='adapters/patient_triage'
)

# Interactive test
while True:
    query = input('\nYou: ')
    if query.lower() in ('quit', 'exit'):
        break
    response = evaluator.generate_response(
        f'[INST] You are a medical triage assistant.\n\n{query} [/INST]',
        max_new_tokens=512
    )
    print(f'\nIMI: {response}')
"
```

### Option B: Run Full Evaluation Suite

```bash
# Evaluate with all metrics + threshold check
python scripts/training/evaluate_adapter.py --adapter patient_triage

# Evaluate specific metrics only
python scripts/training/evaluate_adapter.py \
    --adapter patient_triage \
    --metrics perplexity mcq_accuracy safety

# Evaluate all adapters
for adapter in patient_triage clinical_decision clinical_pharmacist education regulatory_qa research; do
    echo "=== Evaluating ${adapter} ==="
    python scripts/training/evaluate_adapter.py --adapter ${adapter}
done
```
